Prepare a PDF with AWS services used, Linux commands, problems faced, learnings, and total time taken.

## Solution — What I Did Step by Step

### Step 1 — Decided the Content

The PDF report contains the following sections:

1. **Cover page** — title "AWS DevOps Internship Assignment" with submission details (name, college, branch, email, date).
2. **AWS Services Used** — IAM (user, group, policy), EC2 (instance, security group, key pair, EBS volume), Elastic IP.
3. **IAM Setup Details** — how the IAM user and group were created from the root account, policy attached, then switched to IAM user for all work.
4. **Linux Commands Used** — every command grouped task by task, with what each command does.
5. **Problems Faced** — each problem hit, how it was diagnosed, and how it was fixed.
6. **Learnings** — key takeaways from the assignment.
7. **Total Time Taken** — time breakdown per task.
8. **GitHub Repository Link** and live website link.
9. **References** — AWS docs, Red Hat docs, and other resources used.

### Step 2 — Wrote the Report

- Used the detailed notes from each task (Task1.md through Task4.md and Bonus.md) to compile everything into one document.
- Organised commands in a table format with the command and its explanation side by side.
- Listed all three problems (PEM permission, 403 Forbidden / SELinux, dynamic IP change) with the exact fix commands.
- Added the time breakdown for all tasks.

### Step 3 — Exported as PDF

- Wrote the content and exported/saved it as a PDF file.
- The PDF is part of the final submission.
